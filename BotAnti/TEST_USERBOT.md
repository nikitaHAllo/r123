# Запуск и тест Userbot

## 1. Проверь API_ID и API_HASH

Значения `12345678` и `abcdef1234567890...` в примере — **заглушки**. Нужны свои:

1. Зайди на https://my.telegram.org
2. Войди по номеру телефона
3. Открой **API development tools**
4. Создай приложение (если ещё нет) и скопируй **Api id** и **Api hash**
5. Вставь их в `.env`:
   ```
   API_ID=твой_числовой_id
   API_HASH=твой_хеш_без_пробелов
   ```

## 2. Укажи свой ID в ADMINS

Иначе команды `/admin` и `/analyze` не сработают.

**Как узнать свой user ID:**

- Напиши боту [@userinfobot](https://t.me/userinfobot) в Telegram — он пришлёт твой **Id**
- Либо после первого запуска userbot отправь себе в «Избранное» любое сообщение и посмотри логи (если добавим вывод senderId)

В `.env`:
```
ADMINS=123456789
```
(один или несколько через запятую: `123,456,789`)

## 3. Первый запуск (без сессии)

В папке проекта:

```bash
npm run dev
```

Должно появиться:

1. `Введите номер телефона (+7...):` — введи номер в международном формате, например `+79001234567`
2. В Telegram (на этом номере) придёт **код** от Telegram
3. `Введите код из Telegram:` — введи этот код
4. Если включена 2FA: `2FA пароль (если есть):` — введи пароль облачного пароля, иначе просто Enter

После успешного входа в консоль выведется что-то вроде:

```
✅ Вход выполнен. Сохраните сессию в .env:
SESSION_STRING=1BQANOTEuLjE...

Залогинен: Имя
Бот (userbot) запущен 🚀
```

Скопируй **всю** строку `SESSION_STRING=...` и добавь её в `.env`. При следующих запусках ввод телефона и кода не понадобится.

Останови бота (Ctrl+C) и перезапусти:

```bash
npm run dev
```

Теперь должно быть сразу: `✅ Залогинен как userbot: ...` и `Бот (userbot) запущен 🚀`.

## 4. Как тестировать, что всё работает

### 4.1 ЛС (личка) с самим собой

1. Открой **Telegram** на телефоне/десктопе (тот же аккаунт, с которого залогинился userbot).
2. Напиши **себе** в «Избранное» (Saved Messages) или открой диалог с собой через поиск.

**Проверка команд:**

| Действие | Ожидание |
|----------|----------|
| Отправить `/start` | Ответ: «Userbot BotAnti запущен. Команды: /admin, /analyze...» |
| Отправить `/admin` | Сообщение с меню: Удаление, Брань, Реклама, Нейросеть и команды вида `/admin_toggle_*` |
| Отправить `/admin_toggle_profanity` | Ответ вида «✅ Фильтр брани: вкл» или «❌ ... выкл» |
| Отправить `/admin_toggle_ad` | Аналогично для рекламы |

**Проверка фильтров (если включены):**

- Напиши слово из списка брани (например из `PROFANITY_WORDS`: «дурак»).
- Должно прийти сообщение вроде: «Ваше сообщение содержит запрещённый контент. Причина: ненормативная лексика».
- Напиши обычное сообщение без брани — бот не должен отвечать (или только пишет в лог, без ответа в чат).

### 4.2 Группа или канал

1. Создай тестовую группу или используй канал, где ты админ.
2. Напиши в группе/канале сообщение с бранью или ссылкой (если включены фильтры).
3. Ожидание:
   - В консоли или в чат `LOG_CHAT_ID` (если указан) приходит лог о нарушении.
   - Если в `.env` включено удаление (`FILTER_*` и логика удаления), сообщение может быть удалено (userbot удаляет от имени своего аккаунта — нужны права админа в чате).

При необходимости ограничить чаты — укажи в `.env`:

```
ALLOWED_CHATS=-1001234567890,123456789
```

(ID чата/группы можно взять, например, из ссылки или ботов вроде @userinfobot / @getidsbot в нужном чате.)

### 4.3 Файл и анализ нейросетью

1. В ЛС себе отправь **документ** — экспорт чата Telegram (`.json` или `.html`).
2. Должен прийти ответ: «Файл ... загружен! Сообщений: N. Для анализа используйте команду /analyze».
3. Отправь команду `/analyze`.
4. Должно прийти сообщение о старте анализа и потом обновления прогресса; в конце — отчёт (или «нарушений не найдено»).
5. Пока идёт анализ, отправь `/cancel_analysis` — анализ должен остановиться и прийти сообщение вроде «Анализ прерван».

Если на каком-то шаге есть ошибка — смотри вывод в консоли (логи) и текст ошибки.

### 4.4 Краткий чеклист

- [ ] Запуск: `npm run dev` — в консоли «Бот (userbot) запущен»
- [ ] В ЛС: `/start` — приходит приветствие с командами
- [ ] В ЛС: `/admin` — приходит меню
- [ ] В ЛС: `/admin_toggle_profanity` — ответ про вкл/выкл
- [ ] В ЛС: сообщение с бранью — приходит предупреждение (если фильтр вкл)
- [ ] В ЛС: загрузка .json/.html файла — ответ «Файл загружен»
- [ ] В ЛС: `/analyze` — старт анализа и отчёт
- [ ] В ЛС: во время анализа `/cancel_analysis` — анализ останавливается

Если всё из чеклиста выполняется — userbot настроен и работает.
