# Функции: Бот (Grammy) vs Userbot (MTProto)

Кратко: **userbot** только просматривает сообщения в реальном времени, анализирует по промпту (нейросеть) и фильтрам и отсылает нарушения туда, где указано. **Админ-панель и импорт/анализ файлов — только через бота**; настройки в userbot подтягиваются из общей БД.

---

## Сводная таблица

| Функция | Бот (Grammy) | Userbot (MTProto) |
|--------|----------------|-------------------|
| **Модерация в реальном времени** | ✅ В чатах, где бот добавлен и админ | ✅ В чатах из ALLOWED_CHATS (или везде, если пусто) + Избранное |
| **Проверка текста** | Нейросеть + ключевые слова (брань, реклама, кастом) | То же (настройки и темы подтягиваются из общей БД раз в 10 сек) |
| **Удаление сообщений при нарушении** | ✅ Если бот админ с правом удаления и включено «Удаление» | ✅ От имени вашего аккаунта (нужны права админа в чате) |
| **Куда отсылать нарушения** | `LOG_CHAT_ID` (Bot API) | `LOG_CHAT_IDS` (me / ID) + опционально в личку боту (`BOT_USERNAME`) |
| **Загрузка и анализ файлов (HTML/JSON)** | ✅ Документ боту → `/analyze` → клавиатура лимита → отчёт | ❌ Не в userbot (только через бота) |
| **Фильтр по автору при анализе** | ✅ `/filter_author` + кнопки в анализе | — |
| **Админ-панель (меню кнопок)** | ✅ `/admin` в ЛС: Удаление, Брань, Реклама, Нейросеть, Модель, Темы, Статистика, Слова | ❌ Нет; все настройки задаются через бота (общая БД) |
| **Управление словами/темами** | ✅ Команды и кнопки в боте | ❌ Только чтение из БД |
| **Проверка прав в чате** | ✅ `/check_permissions` в группе | ❌ Нет |
| **Режим «проверки в ЛС»** | ✅ `/check_chat`: проверка сообщений админа в ЛС | ❌ Убрано из userbot |
| **Ограничение по чатам** | `ALLOWED_CHATS` для доступа к загрузке/анализу файлов | `ALLOWED_CHATS` — в каких чатах обрабатывать сообщения (Избранное всегда разрешено) |
| **Админы** | По `ADMINS` (user ID) | По `ADMINS` + поддержка `"me"` (текущий аккаунт) |

## Бот (Grammy) — что выполняет

1. **Запуск и админка**
   - `/start` — приветствие; для админов показывается клавиатура.
   - `/admin` — панель только в ЛС: кнопки переключения настроек и переходы в подменю.

2. **Настройки (через панель и callback)**
   - Вкл/выкл: автоудаление, брань, реклама, нейросеть.
   - Выбор модели нейросети, управление темами (вкл/выкл, удаление, добавление).
   - Просмотр статистики и списка слов, справка по командам.

3. **Команды**
   - `/check_chat`, `/stop_check_chat` — включить/выключить режим проверки сообщений в ЛС.
   - `/check_permissions` — в группе проверяет, есть ли у бота права админа (в т.ч. удаление).
   - `/analyze` — после загрузки файлов показывает клавиатуру выбора лимита и запускает анализ.
   - `/filter_author` — фильтр по автору (имя или user_id) для анализа.

4. **Сообщения и документы**
   - В чатах, где бот админ: проверка каждого сообщения (нейросеть + ключевые слова); при нарушении — лог в `LOG_CHAT_ID`, при включённом удалении — удаление сообщения и предупреждение.
   - В ЛС в режиме check_chat: проверка каждого сообщения админа и ответ «нарушение / ок».
   - Документы: приём через Bot API, парсинг HTML/JSON, накопление сообщений для `/analyze`.

5. **Callback-кнопки**
   - Админка: все переключатели и подменю (модели, темы, слова и т.д.).
   - Анализ: выбор лимита, фильтр по автору, список пользователей из файла, отмена анализа.

---

## Userbot (MTProto) — что выполняет

Userbot **разгружен**: только модерация в реальном времени. Админка и анализ файлов — только в боте.

1. **Только реальное время**
   - Просматривает все входящие текстовые сообщения в разрешённых чатах (`ALLOWED_CHATS` или все; Избранное всегда разрешено).
   - Анализирует по промпту (нейросеть, темы из БД) и по фильтрам (брань, реклама, кастом).
   - Нарушения отсылает в `LOG_CHAT_IDS` и при наличии `BOT_USERNAME` — в личку боту; в ЛС пишет пользователю; в группах при включённом удалении — удаляет сообщение.

2. **Настройки**
   - Синхронизация с общей БД раз в 10 сек: `loadSettingsFromDB()`, `loadTopicsFromDB()`. Менять настройки — только через бота.

3. **Что не делает**
   - Не обрабатывает документы (файлы не принимает).
   - Нет команд `/analyze`, `/cancel_analysis`, `/check_chat`, `/stop_check_chat`, `/start`, `/admin`.
   - Нет своей админ-панели.

4. **Прочее**
   - Поддержка `ADMINS` с `"me"`: при старте подставляется ID текущего аккаунта.
   - Действует как пользователь/админ в чате (удаление от своего имени).

---

## Общее (и бот, и userbot)

- Одна и та же логика проверки: нейросеть (если включена) + фильтры по ключевым словам (брань, реклама, кастом).
- Общая БД: настройки, слова, темы, статистика.
- Оба могут удалять сообщения при нарушении (бот — если добавлен админом с правом удаления, userbot — если у аккаунта есть права в чате).
- Анализ экспорта чата (HTML/JSON) и отчёт по нарушениям — только в боте (загрузка файла боту → `/analyze`).
