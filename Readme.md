## 🔧 Установка

### Требования

- Node.js 18+ 
- npm или yarn
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))
- Доступ к API нейросети (Ollama или совместимый сервис)

### Шаги установки

1. **Клонируйте репозиторий**:
```bash
git clone <repository-url>
cd BotAnti
```

2. **Установите зависимости**:
```bash
npm install
```

3. **Создайте файл `.env`** в корне проекта (см. раздел [Конфигурация](#конфигурация))

4. **Соберите проект**:
```bash
npm run build
```

5. **Запустите бота**:
```bash
npm start
```

Для разработки используйте:
```bash
npm run dev
```

## ⚙️ Настройка

### Добавление бота в чат

1. Пригласите бота в группу или канал через поиск по username
2. Назначьте бота администратором
3. Обязательно выдайте права:
   - ✅ **Удаление сообщений** (обязательно)
   - ✅ Блокировка пользователей (опционально)

Используйте команду `/check_permissions` в группе для проверки прав бота.

## 🔐 Конфигурация

Создайте файл `.env` в корне проекта со следующим содержимым:

```env
# Обязательные параметры
BOT_TOKEN=your_bot_token_here

# Администраторы (ID через запятую)
ADMINS=123456789,987654321

# Разрешенные чаты для анализа файлов (ID через запятую, пусто = все)
ALLOWED_CHATS=123456789,-1001234567890

# ID чата для логирования нарушений (опционально)
LOG_CHAT_ID=-1001234567890

# Фильтры (true/false)
FILTER_PROFANITY=true
FILTER_ADVERTISING=true

# Начальные слова для фильтров (через запятую)
PROFANITY_WORDS=слово1,слово2,слово3
AD_KEYWORDS=реклама,спам,купить
```

### Получение Chat ID

Для получения ID чата используйте утилиту:
```bash
npx tsx src/get_group_id.ts
```

Отправьте любое сообщение боту в нужном чате, и он вернет Chat ID.

## 📖 Использование

### Базовое использование

После настройки бот автоматически начинает анализировать все сообщения в чатах, где он является администратором.

### Режимы работы

1. **Автоматическая модерация**: Бот проверяет все сообщения в реальном времени
2. **Режим проверки ЛС**: Администратор может включить режим `/check_chat` для тестирования фильтров в личных сообщениях
3. **Анализ файлов**: Загрузите экспортированный чат (HTML/JSON) для массового анализа

## 🎮 Команды

### Команды для всех пользователей

- `/start` - Запуск бота и приветствие
- `/analyze` - Начать анализ загруженных файлов (требуются права администратора)

### Команды администратора

#### Управление панелью
- `/admin` - Открыть админ-панель (только в ЛС)
- `/check_chat` - Включить режим проверки личных сообщений
- `/stop_check_chat` - Выключить режим проверки
- `/check_permissions` - Проверить права бота в группе

#### Управление словами
- `/add_profanity <слово>` - Добавить слово в фильтр бранных слов
- `/del_profanity <слово>` - Удалить слово из фильтра бранных слов
- `/add_ad <слово>` - Добавить слово в фильтр рекламы
- `/del_ad <слово>` - Удалить слово из фильтра рекламы
- `/add_custom <слово>` - Добавить пользовательское слово
- `/del_custom <слово>` - Удалить пользовательское слово

#### Управление нейросетью
- `/test_neural <текст>` - Протестировать нейросеть на тексте
- `/models` - Показать список доступных моделей
- `/neural_stats` - Показать статистику работы нейросети

#### Управление тематиками
- `/add_topic <имя> | <описание> | <приоритет> | <prompt>` - Добавить новую тематику
  - Пример: `/add_topic cars | Автомобильная тема | 1 | Твой кастомный prompt`
- `/del_topic <имя>` - Удалить тематику

## 🔍 Фильтры

### Типы фильтров

1. **Фильтр бранных слов** (`FILTER_PROFANITY`)
   - Проверяет наличие запрещенных слов в сообщении
   - Слова добавляются через команды или `.env`

2. **Фильтр рекламы** (`FILTER_ADVERTISING`)
   - Обнаруживает рекламные ключевые слова
   - Настраивается администратором

3. **Пользовательские слова** (`custom_words`)
   - Произвольные слова, добавленные администратором
   - Всегда активны

### Приоритет проверки

1. Сначала проверяется нейросеть (если включена)
2. Затем проверяются фильтры по ключевым словам
3. При обнаружении нарушения обработка прекращается

## 🧠 Нейросеть

### Настройка API

Бот использует Ollama API (или совместимый сервис). URL настраивается в `src/neural.ts`:

```typescript
const NEURAL_API_URL = 'http://10.8.0.24:11434/v1/chat/completions';
```

### Доступные модели

- `qwen2.5-coder:7b`
- `qwen3:30b`
- `hf.co/bartowski/Qwen_Qwen3-30B-A3B-Thinking-2507-GGUF:Q4_K_M`
- `hf.co/unsloth/Qwen3-30B-A3B-Instruct-2507-GGUF:Q4_K_M`

### Тематики

Тематики определяют, какие типы контента должна обнаруживать нейросеть. Каждая тематика имеет:
- **Название** - уникальный идентификатор
- **Описание** - описание того, что должна обнаруживать тема
- **Приоритет** - порядок проверки (меньше = выше приоритет)
- **System prompt** - промпт для нейросети
- **Статус** - включена/выключена

### Добавление тематики

```
/add_topic название | описание | приоритет | кастомный_prompt
```

Пример:
```
/add_topic cars | Сообщения об автомобилях | 1 | Ты анализируешь сообщения на тему автомобилей
```

Если кастомный prompt не указан, используется стандартный шаблон.

### Принцип работы

1. Нейросеть получает сообщение и system prompt тематики
2. Возвращает числовое значение уверенности (0-100)
3. Если уверенность > 80%, нарушение считается обнаруженным
4. Проверка останавливается при первом обнаружении

## 📄 Анализ файлов

### Поддерживаемые форматы

- **HTML** - Экспорт чата из Telegram Desktop
- **JSON** - Экспорт чата из Telegram Desktop

### Процесс анализа

1. **Загрузка файла**: Отправьте файл боту в личном сообщении или группе
2. **Обработка**: Бот извлекает все сообщения из файла
3. **Выбор количества**: Используйте `/analyze` для выбора количества сообщений для анализа
4. **Анализ**: Бот последовательно анализирует каждое сообщение
5. **Отчет**: Получите подробный отчет о найденных нарушениях

### Ограничения анализа

- Можно выбрать: все сообщения, 500, 1000, 2000, 5000, 10000 или ввести свое число
- Анализ можно отменить в любой момент
- Прогресс отображается в реальном времени

## 📊 Статистика

### Типы нарушений

- `violation_profanity` - Ненормативная лексика
- `violation_ad` - Реклама
- `violation_custom` - Запрещенные слова
- `neural_bad_words` - Нежелательный контент (нейросеть)
- `neural_cars` - Автомобильная тема (нейросеть)
- `neural_advertising` - Реклама (нейросеть)

### База данных

Статистика хранится в SQLite базе данных `database.db`:
- Таблица `statistics` - все события
- Таблица `profanity_words` - слова брани
- Таблица `ad_keywords` - рекламные слова
- Таблица `custom_words` - пользовательские слова
- Таблица `topics` - тематики нейросети

- **SQLite** - база данных
- **Axios** - HTTP клиент для API нейросети
- **Cheerio** - парсинг HTML файлов

### Добавление новых фильтров

1. Добавьте функцию проверки в `src/filters.ts`
2. Добавьте обработку в `src/handlers/messageHandler.ts`
3. Добавьте тип нарушения в `src/handlers/violationHandler.ts`
4. Обновите админ-панель в `src/admin.ts`

### Добавление новых тематик нейросети

Используйте команду `/add_topic` или добавьте напрямую в базу данных:

```sql
INSERT INTO topics (name, description, system_prompt, priority, enabled)
VALUES ('название', 'описание', 'prompt', 1, 1);
```

## ⚠️ Важные замечания

1. **Права бота**: Бот должен быть администратором с правом удаления сообщений
2. **Логирование**: Настройте `LOG_CHAT_ID` для отслеживания нарушений
3. **Производительность**: Нейросетевой анализ может быть медленным для больших объемов
4. **База данных**: Регулярно делайте резервные копии `database.db`
5. **API нейросети**: Убедитесь, что API доступен и работает стабильно
